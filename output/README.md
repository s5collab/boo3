# output/

This directory is intentionally empty in the repository. All contents here are
intermediate products of the analysis pipeline (GMM chains, per-component
orbital trajectories, particle-spray caches, action/peri-apo caches) and are
regenerated by running the analysis scripts in `notebooks/`:

1. `python notebooks/01_compute_gmm.py`        → `boo3_gmm_samples.h5`, `boo3_gmm_membership.csv`
2. `python notebooks/01_compute_orbits.py`     → `orbit_McMillan17_samples.npz`, ..._trajectories.npz, `boo3_21mem_vgrad_samples.h5`
3. `python notebooks/02_compute_spray.py all`  → `spray_cache_new6D/spray_<key>.pkl`
4. `jupyter nbconvert --execute notebooks/01_members_gmm_orbits.ipynb` → `boo3_rrl_distances.npz`
5. `jupyter nbconvert --execute notebooks/04_action_peri_sgr_fig9_10_11.ipynb` → `nb4_cache/lvdb_dwarfs_actions_mc50.pkl`, etc.

Approximate wall-clock for a full regeneration on a modern laptop: ~3–4 hours
(the spray-cache step dominates at ~2 h).
